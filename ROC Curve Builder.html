Overview

ROC Curve Builder is a free, web-based R-Shiny application designed for researchers, data scientists, and clinicians who need to evaluate binary classification models. It provides interactive ROC (Receiver Operating Characteristic) curve visualization, comprehensive performance metrics, confusion matrix output, and professional export capabilities — all without requiring any programming knowledge.

Problems ROC Curve Builder Solves

📈
ROC Curve VisualizationGenerates publication-ready ROC curves from uploaded datasets with AUC annotation and shaded area-under-curve.
🧮
Performance MetricsCalculates AUC, accuracy, sensitivity, and specificity with confusion matrix output in a single click.
🔄
Variable RecodingRecodes non-binary outcome variables to 0/1 directly within the app — no pre-processing required.
📄
Professional ReportingDownloads high-resolution PNG plots and comprehensive Word (.docx) reports with metrics and analysis details.

Local Installation

Step 1 — Download app.R

Download the source file from the GitHub repository: github.com/mudassiribrahim12

Step 2 — Install required packages

Run the following in your R console:

install.packages(c(
  "shiny", "ggplot2", "pROC",
  "MLmetrics", "DT", "caret",
  "dplyr", "readxl", "haven",
  "shinythemes", "officer",
  "shinycssloaders", "shinyjs"
))

Step 3 — Load the libraries

library(shiny)
library(ggplot2)
library(pROC)
library(MLmetrics)
library(DT)
library(caret)
library(dplyr)
library(readxl)
library(haven)
library(shinythemes)
library(officer)
library(shinycssloaders)
library(shinyjs)

Step 4 — Set working directory and run

Replace the path below with the location of your saved app.R file:

# Windows
setwd("C:/Users/YourName/Desktop/")

# Linux / macOS
setwd("/home/yourname/Desktop/")

# Launch the app
library(shiny)
runApp("app.R")

Step 5 — Open in browser

The app opens in the RStudio viewer pane. Click Open in Browser for the best experience, especially for plot rendering.

Deployment

Deploy your own instance to shinyapps.io using the rsconnect package.

# Install rsconnect
install.packages("rsconnect")

# Deploy — replace with actual path to your app folder
rsconnect::deployApp("path/to/ROC-Curve-Builder")
⚠️ Note

Replace "path/to/ROC-Curve-Builder" with the actual folder path containing your app.R file.

Analysis Workflow

Follow these steps inside the ROC Curve Builder to perform a complete binary classification performance analysis.

1
Navigate to "Build ROC Curve"

Click the Build ROC Curve tab in the top navigation bar, or click the Start Analysis button on the Home page.

2
Upload Your Dataset

Click Browse Files… and select your file. Supported formats: CSV, Excel (.xlsx/.xls), Stata (.dta), SPSS (.sav). The app displays a confirmation message showing the number of rows and columns loaded.

3
Select Variables

Choose your Outcome Variable (binary, must be coded 0/1) and one or more Predictor Variables. A warning appears if the outcome has more than two levels.

4
Recode Outcome if Needed

If your outcome is not already 0/1 (e.g., "Yes"/"No" or "Disease"/"No Disease"), use the recode controls to map original values to 0 and 1 without leaving the app.

5
Customize the Plot

Set the graph title, axis labels, color palette, line thickness, and transparency. Toggle AUC annotation and the diagonal reference line on or off.

6
Generate ROC Curve

Click Generate ROC Curve. The app fits a logistic regression model, calculates predicted probabilities, computes the ROC curve, and displays performance metrics and the confusion matrix.

7
Download Results

Click Download Plot (PNG) for a 300 dpi high-resolution image, or Download Report (DOCX) for a Word document containing the plot, metrics, and analysis details.

Data Input & Formats

ROC Curve Builder reads data files directly from your computer. No data is stored or transmitted externally — all processing occurs locally within your R session.

FormatExtensionR Function UsedNotes
CSV.csvread.csv()UTF-8 encoding first; falls back to Latin-1 on error

SPSS.savread_sav()Variable labels preserved
Stata.dtaread_dta()Variable labels preserved
📋 Data Requirements

Your dataset must contain at least one binary outcome column and one numeric predictor column. Rows with missing values in the selected variables are automatically removed via na.omit() before analysis.

⚠️ Outcome Variable

The outcome variable must be coded 0 and 1 before running the analysis. Use the built-in recoding controls if your outcome uses other labels. The app warns you if more than two unique values are detected.

Variable Recoding

The recoding panel appears automatically once an outcome variable is selected. It allows you to map two original values to 0 and 1 without modifying your source file.

1
Select Outcome Variable

Choose the outcome column from the dropdown. The recode panel populates with the unique values found in that column.

2
Map Original Value 1 → 0

In Original Value 1, select the value that should represent the negative class (e.g., "No", "Control", "Negative"). In Recode to, enter 0.

3
Map Original Value 2 → 1

In Original Value 2, select the value that should represent the positive class (e.g., "Yes", "Case", "Positive"). In Recode to, enter 1.

4
Run Analysis

Recoding is applied internally before model fitting. Your original file is not modified.

Performance Metrics

After clicking Generate ROC Curve, the following metrics are computed from a logistic regression model and displayed in the metrics table.

AUC

Area under the ROC curve

Ranges from 0 to 1. Values > 0.7 indicate acceptable discrimination; > 0.8 is excellent. A value of 0.5 indicates no discrimination (random classifier).

Accuracy

(TP + TN) / (TP + TN + FP + FN)

Proportion of all predictions that are correct. Can be misleading for imbalanced datasets.

Sensitivity (Recall)

TP / (TP + FN)

Proportion of actual positives correctly identified. Also called the True Positive Rate (TPR) — the Y-axis of the ROC curve.

Specificity

TN / (TN + FP)

Proportion of actual negatives correctly identified. 1 − Specificity = False Positive Rate (FPR) — the X-axis of the ROC curve.

Confusion Matrix

The confusion matrix displays the counts of true negatives (TN), false positives (FP), false negatives (FN), and true positives (TP) at the default classification threshold of 0.5.

Actual 0 (Negative)Actual 1 (Positive)
Predicted 0TN — True NegativeFN — False Negative (Type II Error)
Predicted 1FP — False Positive (Type I Error)TP — True Positive
📐 Model Used

ROC Curve Builder fits a binary logistic regression model (glm(..., family = "binomial")) using the selected predictor(s). Predicted probabilities are used to construct the ROC curve via the pROC package. The classification threshold for the confusion matrix is fixed at 0.5.

Assumptions

⚖️
Binary Outcome

The outcome variable must have exactly two classes (0 = negative, 1 = positive). Multi-class outcomes are not supported.

📊
Numeric Predictors

Predictor variables should be numeric. Categorical predictors require manual dummy coding before upload.

🔗
Independent Observations

Logistic regression assumes observations are independent. Clustered or longitudinal data requires more advanced methods.

📈
Both Classes Present

The dataset must contain observations from both class 0 and class 1 after removing missing values.

🔢
Adequate Sample Size

Logistic regression and ROC analysis require sufficient events in both classes for reliable estimation.

Limitations

🔄
Fixed Classification Threshold

The confusion matrix uses a fixed threshold of 0.5. Optimal threshold selection (e.g., Youden's J) is not currently implemented.

📉
Logistic Regression Only

The app uses logistic regression to generate predicted probabilities. Other classifiers (Random Forest, SVM, XGBoost) are not supported natively.

⚠️
No Cross-Validation

ROC metrics are computed on the training data, not a held-out test set. This may lead to optimistic performance estimates. External validation is recommended.

🧮
No Confidence Intervals

AUC confidence intervals and DeLong's test for comparing ROC curves are not displayed in the current version.

🌐
Session-Based Only

Results are not saved between sessions. Download your report before closing the browser.
