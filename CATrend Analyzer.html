CATrend Analyzer

A Tool for Performing the Cochran-Armitage Test for Trend

CATrend Analyzer is a free, web-based R-Shiny application designed specifically for detecting trends in proportions across ordered categorical variables. This test is widely used in clinical trials, epidemiology, and public health research, particularly when assessing whether the probability of a binary outcome changes consistently across increasing levels of an exposure or dose.

Before You Begin:
  • Ensure your data is clean with no missing values (or be aware that rows with missing values will be removed)
  • Variable names should not contain spaces or special characters
  • For binary dependent variables, ensure they are coded as 0/1 or two categories
  • Have at least 5 observations per cell for reliable estimates

Statistical Formula

The Cochran-Armitage test for trend evaluates whether there is a linear trend in proportions across ordered groups. The application implements the exact formula:

Z = [Σ yj (xj - x̄)] / √[p̄(1-p̄) Σ nj (xj - x̄)²]

Where:

  • xj = scores assigned to the ordered groups (1, 2, 3, ..., k)
  • yj = number of "successes" (events) in group j
  • nj = total number of observations in group j
  • x̄ = weighted mean of scores = Σ(njxj) / Σnj
  • p̄ = overall proportion of successes = Σyj / Σnj
  • k = number of ordered groups

The test statistic Z follows a standard normal distribution. The chi-square equivalent is Z² with 1 degree of freedom.

Continuity Correction: For small samples, the app can apply a continuity correction of c = 0.5 × (score spacing) to improve approximation accuracy. This adjusts the numerator by ±c based on the test direction.

Test Interpretation:

  • Z > 0: Increasing trend (proportion of successes increases across ordered groups)
  • Z < 0: Decreasing trend (proportion of successes decreases across ordered groups)
  • p-value < 0.05: Statistically significant trend at α = 0.05

Live Application

No installation required: Use the web version directly in your browser.

Prerequisites

  • R version 4.4 or higher (for local installation)
  • RStudio (recommended for local development)
  • A modern web browser (for web version)

Steps to Run Locally

1. Download app.R from GitHub
Access the source code at: https://github.com/mudassiribrahim12/CATrend-Analyzer/blob/main/app.R

2. Install required R packages

# Install required packages
install.packages(c(
  "shiny", "shinydashboard", "DT", "haven", "readxl", "ggplot2",
  "officer", "rmarkdown", "tidyverse", "sortable", "scales"
))

3. Load required R packages

# Load libraries
library(shiny)

library(haven)
library(readxl)

library(officer)

4. Run the application in RStudio

# Set your working directory to where you saved app.R
setwd("C:/Users/YourName/Desktop/")    # Windows example
# or
setwd("/home/yourname/Desktop/")       # Linux/Mac example

library(shiny)

5. Open the application in a web browser

Complete Analysis Workflow - Step-by-Step Guide

Follow these steps to perform a complete trend analysis from data upload to report download.

1
Select Data Type
Choose either "Normal Data" (individual-level dataset) or "Crosstabulation Data" (pre-summarized count matrix). For Normal Data, upload CSV, Excel, Stata (.dta), or SPSS (.sav) files. For Crosstabulation Data, you will define variable levels and enter counts in an interactive table.
2
Upload Your Data File (Normal Data Mode)
Select file type (CSV, Excel, Stata, SPSS) and browse to upload your data. For CSV files, customize delimiter selection, header presence, and quote characters. For Excel files, specify sheet number and cell range. For Stata and SPSS files, the application automatically detects haven_labelled variables and converts them to factors with appropriate labels.
3
Setup Crosstabulation Table (Crosstabulation Mode)
Specify names for dependent and independent variables, and define the number of levels for each variable. Click "Setup Crosstabulation Table" to generate an interactive table where you provide level names and enter counts for each cell. Then click "Create Dataset from Crosstabulation" to reconstruct the full individual-level dataset by expanding the counts into individual observations.
4
Review Missing Data & Data Quality
Check missing data summaries showing missing counts and percentages for each variable. Use the "Remove Rows with Missing Values in Selected Variables" button to clean the data. Review the data quality report displaying dataset dimensions, complete case counts, variable types, and unique value counts to identify potential issues.
5
Select Variables
Choose the dependent variable (binary outcome) and the independent variable (ordered exposure or grouping variable) from dropdown menus that display all available variables in the cleaned dataset.
6
Order Independent Variable Levels
Use the drag-and-drop bucket lists to arrange levels from lowest to highest. Simply drag each level and drop it before or after another level to create a meaningful order (e.g., Placebo → Low → Medium → High). Dependent variable levels can also be ordered, with the first level serving as the reference category for multi-category outcomes. Click "Update Level Ordering" to register the specified ordering.
7
Configure Test Options
Select test type: two-sided (testing for any trend direction), increasing, or decreasing. Optionally apply continuity correction for small samples.
8
Run the Cochran-Armitage Test
Click "Run Cochran-Armitage Test" to generate results. The application returns the Z-statistic, chi-square statistic, p-value, and detailed intermediate calculations for validation.
9
Customize Visualization
Create customizable stacked bar plots displaying counts, row percentages, or column percentages. Adjust title size, label size, legend size, and axis text sizes. Assign custom colors to dependent variable levels using the interactive color picker interface.
10
Download Full Report
Click "Download Full Report" to obtain a Word document containing the complete analysis, including the customized graph, statistical test results, and detailed interpretation, suitable for inclusion in manuscripts or research records.

Assumptions of the Cochran-Armitage Test

  • Independence: Observations are independent of each other
  • Ordered Categories: Independent variable levels have a natural order
  • Monotonic Trend: The trend is assumed to be monotonic
  • Adequate Sample Size: Expected cell counts should ideally be >5
  • Binary Outcome: The test is designed for binary dependent variables

Limitations

  • Small Sample Sizes: Results may be unreliable when expected cell counts < 5
  • Sparse Data: Many zero cells can affect test accuracy
  • Non-monotonic Trends: The test may not detect non-linear patterns
  • Confounding: The test does not adjust for other variables
