CATrend Analyzer

A Specialized Tool for Performing the Cochran-Armitage Test for Trend

CATrend Analyzer is a free, web-based R-Shiny application designed specifically for detecting trends in proportions across ordered categorical variables. This test is widely used in clinical trials, epidemiology, and public health research, particularly when assessing whether the probability of a binary outcome changes consistently across increasing levels of an exposure or dose.

Before You Begin:
  • Ensure your data is clean with no missing values (or be aware that rows with missing values will be removed)
  • Variable names should not contain spaces or special characters
  • For binary dependent variables, ensure they are coded as 0/1 or two categories
  • Have at least 5 observations per cell for reliable estimates

Statistical Formula

The Cochran-Armitage test for trend evaluates whether there is a linear trend in proportions across ordered groups. The application implements the exact formula:

Z = [Σ yj (xj - x̄)] / √[p̄(1-p̄) Σ nj (xj - x̄)²]

Where:

  • xj = scores assigned to the ordered groups (1, 2, 3, ..., k)
  • yj = number of "successes" (events) in group j
  • nj = total number of observations in group j
  • x̄ = weighted mean of scores = Σ(njxj) / Σnj
  • p̄ = overall proportion of successes = Σyj / Σnj
  • k = number of ordered groups

The test statistic Z follows a standard normal distribution. The chi-square equivalent is Z² with 1 degree of freedom.

Continuity Correction: For small samples, the app can apply a continuity correction of c = 0.5 × (score spacing) to improve approximation accuracy. This adjusts the numerator by ±c based on the test direction.

Test Interpretation:

  • Z > 0: Increasing trend (proportion of successes increases across ordered groups)
  • Z < 0: Decreasing trend (proportion of successes decreases across ordered groups)
  • p-value < 0.05: Statistically significant trend at α = 0.05

Complete Analysis Workflow - Step-by-Step Guide

Follow these steps in order to perform a complete trend analysis from data upload to report download.

1
Select Data Type
  • In the Data Upload tab, choose either "Normal Data" (raw dataset) or "Crosstabulation Data" (pre-summarized counts)
  • For Normal Data: Each row represents one observation
  • For Crosstabulation Data: Enter counts directly in a contingency table format
2
Upload Your Data File (Normal Data Mode)
  • Select file type: CSV, Excel, Stata, or SPSS
  • Click "Browse" and select your data file
  • For CSV: specify separator, header, and quote settings
  • For Excel: specify sheet number and optional cell range
  • Wait for data preview to appear in the "Data Preview" table
3
Enter Crosstabulation Data (Crosstab Mode)
  • Enter Dependent Variable Name and Independent Variable Name
  • Set number of levels for both variables
  • Click "Setup Crosstabulation Table"
  • Enter labels for each level and counts for each cell
  • Click "Create Dataset from Crosstabulation" to expand counts into observations
4
Review Missing Data
  • Go to the Data Cleaning tab
  • Review the missing data summary table showing missing values by variable
  • Click "Remove Rows with Missing Values in Selected Variables" to clean data
  • Check the cleaned data preview to confirm removal of missing cases
5
Select Variables for Analysis
  • Navigate to the Variable Selection tab
  • Dependent Variable: Select your binary or categorical outcome variable
  • Independent Variable: Select your ordered exposure or grouping variable
  • Review the variable information panel to confirm selections
6
Order Independent Variable Levels
  • In the "Order Independent Variable Levels" section, locate the bucket list
  • Drag levels from the left bucket to the right bucket in the correct order
  • Place LOWEST level at the TOP and HIGHEST level at the BOTTOM
  • Example: Placebo → Low Dose → Medium Dose → High Dose
  • Click "Update Level Ordering" to save the order
  • Verify the order appears correctly in the "Current Variable Information" panel
Critical: Correct level ordering is essential for accurate trend detection. The order determines whether the test detects an increasing or decreasing trend.
7
Order Dependent Variable Levels (Optional)
  • If your dependent variable has more than 2 levels, you can reorder them
  • The first level will be treated as the "success" category for the trend test
  • For binary variables, ordering does not affect the test results
  • Click "Update Level Ordering" after making changes
8
Configure Test Options
  • Go to the Analysis tab
  • Test Type: Choose one:
    • Two-sided: Detects any trend (increasing or decreasing)
    • One-sided (increasing): Tests if proportion increases across levels
    • One-sided (decreasing): Tests if proportion decreases across levels
  • Continuity Correction: Check this box to apply continuity correction for small samples (recommended)
9
Run the Cochran-Armitage Test
  • Click the "Run Cochran-Armitage Test" button
  • Wait for the analysis to complete (a notification will appear)
  • The test results will appear in the "Cochran-Armitage Test Results" section
10
Review Test Results
  • Check the Z-statistic (positive = increasing trend, negative = decreasing trend)
  • Check the p-value (p < 0.05 indicates statistical significance)
  • Review the Enhanced Crosstabulation Results showing counts, row %, column %, and total %
  • Examine the detailed test calculations including numerator, variance term, and intermediate values
  • Read the interpretation section for guidance on your results
Interpretation Example:
"The Cochran-Armitage test revealed a significant increasing trend in disease prevalence across dose levels (Z = 4.12, p < 0.001)."
11
Customize Visualization
  • In the Plot Customization section, select Plot Value Type:
    • Counts: Raw frequencies
    • Row Percentages: Percentage within each independent variable level
    • Column Percentages: Percentage within each dependent variable level
  • Set Plot Title, Title Size, and other text sizes
  • Click on color pickers to customize colors for each dependent variable level
  • The Enhanced Stacked Bar Plot will update automatically with your settings
12
Download Full Analysis Report
  • Navigate to the Results tab
  • Review the report preview showing key findings
  • Select "Word Document" as the report format
  • Click the "Download Full Report" button
  • The report will include:
    • Complete test results with Z-statistic and p-value
    • Enhanced contingency table with counts and percentages
    • Customized stacked bar plot with your chosen settings
    • Detailed interpretation of findings
    • Methodological details and assumptions
  • The report will save to your computer as a Word document (.docx)
Note: The report includes all analysis outputs, the customized plot, and statistical interpretation. You can share this report directly with colleagues or include it in your research.

Assumptions of the Cochran-Armitage Test

  • Independence: Observations are independent of each other
  • Ordered Categories: Independent variable levels have a natural order
  • Monotonic Trend: The trend is assumed to be monotonic (consistently increasing or decreasing)
  • Adequate Sample Size: Expected cell counts should ideally be >5 for reliable results
  • Binary Outcome: The test is designed for binary dependent variables (though it can be adapted for multi-category by specifying a reference level)
Checking Assumptions:
  • Review the contingency table for small cell counts (values less than 5)
  • Ensure independent variable levels are correctly ordered from lowest to highest
  • Verify independence (no repeated measures or clustering in your study design)

Limitations

  • Small Sample Sizes: Results may be unreliable when expected cell counts < 5
  • Sparse Data: Many zero cells can affect test accuracy and power
  • Multi-category Outcomes: Test is designed for binary outcomes; multi-category requires specifying a reference level, which may not capture all patterns
  • Non-monotonic Trends: The test may not detect non-linear patterns (e.g., U-shaped or J-shaped relationships)
  • Confounding: The test does not adjust for other variables (unadjusted analysis only)
