MedModr

An Open-Source Mediation & Moderation Software for Structural Equation Modeling

MedModr is a free, web-based R-Shiny application that provides an intuitive interface for conducting mediation and moderation analyses. Built on the powerful lavaan package, it enables researchers to test complex statistical models without requiring advanced programming skills. The tool supports simple mediation, serial mediation, moderation, and moderated mediation models with professional visualization and report generation.

Common research problems solved:
  • Complex mediation pathways requiring advanced statistical knowledge
  • Difficulty testing moderated mediation (conditional process models)
  • Manual calculation of indirect effects and standard errors
  • Lack of publication-ready diagrams and tables

MedModr solves these problems by automatically generating lavaan syntax, computing indirect effects with bootstrap confidence intervals, and producing customizable path diagrams with coefficient estimates.

Key Features

  • Dual Interfaces: Interactive point-and-click OR syntax-based model specification
  • Automatic Syntax Generation: The app generates proper lavaan syntax based on your variable selections
  • Bootstrap Confidence Intervals: Robust standard errors for indirect effects
  • Comprehensive Fit Measures: Chi-square, CFI, TLI, RMSEA, SRMR
  • Customizable Path Diagrams: Display standardized coefficients with p-values
  • Simple Slopes Visualization: Plot conditional effects for moderation analysis
  • Professional Word Reports: Download analysis results with tables and interpretation
  • Covariate Adjustment: Control for confounding variables (categorical or continuous)
  • Dark/Light Theme Toggle: User preference saved locally

Installation and Local Setup

Steps to Run Locally

1. Download app.R from GitHub
Access the source code at: https://github.com/mudassiribrahim12/MedModr/blob/main/app.R

2. Install required R packages

# Install required packages
install.packages(c(
  "shiny", "lavaan", "semPlot", "ggplot2", "readr", "readxl",
  "haven", "DT", "colourpicker", "officer", "magick", "flextable",
  "shinythemes", "shinyjs", "shinydashboard"
))

3. Run the application in RStudio

# Set your working directory to where you saved app.R
# Example: if app.R is on your Desktop
setwd("C:/Users/YourName/Desktop/")    # Windows example
# or
setwd("/home/yourname/Desktop/")       # Linux/Mac example

# Load the shiny library
library(shiny)

# Run the application
runApp("app.R")
Note: Replace the path with the actual location where you saved the app.R file. Alternatively, open the file in RStudio and click the "Run App" button.

Dual Interfaces

MedModr offers two ways to specify your model:

Interactive Interface
Point-and-click variable selection. Perfect for beginners and quick analyses. Simply select variables from dropdown boxes and let the app generate the lavaan syntax automatically.
Syntax Interface
Write lavaan syntax directly for maximum flexibility. Ideal for advanced users who want complete control over model specification.

Simple Mediation Analysis

Test whether the effect of an independent variable (X) on a dependent variable (Y) is transmitted through a mediator (M).

Model Specification: X → M → Y

Interactive Mode Syntax

# Simple Mediation Model
M ~ a*X
Y ~ b*M + c*X

# Indirect and total effects
Indirect_Effect_M := a*b
Total_Effect := c + (a*b)

Example with Real Variables

# Example: Stress → Anxiety → Performance
Anxiety ~ a*Stress
Performance ~ b*Anxiety + c*Stress

Indirect_Effect := a*b
Total_Effect := c + (a*b)

Serial Mediation Analysis

Test sequential pathways where mediators are ordered (X → M1 → M2 → Y).

Model Specification: X → M1 → M2 → Y 
# Serial Mediation Model
M1 ~ a1*X
M2 ~ a2*M1 + d*X
Y ~ b1*M1 + b2*M2 + c*X

# User-friendly indirect effects
Indirect_Effect_M1 := a1 * b1
Indirect_Effect_M2 := d * b2
Indirect_Effect_Serial := a1 * a2 * b2
Total_Indirect_Effect := Indirect_Effect_M1 + Indirect_Effect_M2 + Indirect_Effect_Serial
Total_Effect := c + Total_Indirect_Effect

Moderation Analysis

Test whether the effect of X on Y depends on the level of a moderator (W).

Automatic Interaction Term: The app automatically creates centered variables and the interaction term when you select variables for moderation analysis. 
# Moderation Model
Y ~ b1*X + b2*W + b3*X_W

# Simple slopes (W centered at -1 SD, mean, +1 SD)
Simple_Slope_Low := b1 + b3*(-1)
Simple_Slope_Avg := b1 + b3*(0)
Simple_Slope_High := b1 + b3*(1)

Moderated Mediation (Conditional Process Model)

Test whether the indirect effect of X on Y through M depends on the level of a moderator W (path a moderation).

Model Specification: Path a (X → M) is moderated by W 
# Moderated Mediation Model
M ~ a1*X + a2*W + a3*X_W
Y ~ b*M + cp*X

# Conditional indirect effects at different levels of W
Conditional_Indirect_Low := (a1 + a3*(-1)) * b
Conditional_Indirect_Avg := (a1 + a3*(0)) * b
Conditional_Indirect_High := (a1 + a3*(1)) * b

# Index of Moderated Mediation
Index_Moderated_Mediation := a3 * b

Path Diagrams

Visualize your model with customizable diagrams that display standardized coefficients and p-values.

  • Layout options: tree, circle, spring, tree2, tree3
  • Customizable colors: observed and latent variable colors
  • Node labels: Customize variable names displayed on the diagram
  • Edge labels: Display standardized coefficients with significance indicators
  • Download options: High-resolution PNG with mediation effects annotation
Diagram Features: Path diagrams automatically display standardized coefficients with p-values, and for mediation analyses, effect estimates are annotated directly on the diagram.

Simple Slopes Plot

For moderation analysis, visualize the conditional effect of X on Y at different levels of the moderator W.

  • Plot shows regression lines at low (-1 SD), average (mean), and high (+1 SD) moderator levels
  • Customizable colors, line widths, and point transparency
  • Interactive interpretation panel with slope values
  • Download as high-resolution PNG for publication

Statistical Engine: lavaan

MedModr uses the lavaan (Latent Variable Analysis) package for all structural equation modeling analyses. Key features:

  • Maximum Likelihood (ML) estimation as default
  • Bootstrap support for robust standard errors and confidence intervals (up to 5,000 samples)
  • Comprehensive fit measures: χ², CFI, TLI, RMSEA, SRMR
  • Standardized solutions (std.all) for effect size interpretation
  • User-defined parameters using := operator for indirect effects

Current Limitations

  • Latent Variables: The current version focuses on observed variable models. Latent variable SEM will be added in future releases.
  • Complex Models: Very large datasets (>10,000 rows) are automatically truncated for performance.
  • Missing Data: Listwise deletion is used. Multiple imputation is not currently supported.
  • Bootstrap Samples: Limited to 5,000 samples for performance reasons.
  • File Size: Performance depends on system resources. Very large datasets may require longer processing times.
